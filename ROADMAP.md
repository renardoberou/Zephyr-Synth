# Zephyr Synth Target Roadmap

This project is no longer treated as a loose feature chain. The target is a coherent **premium MPE performance synth** with a multi-view shell inspired by the current reference concept.

## Target Product Shape

Zephyr should converge toward four integrated workspaces:

1. **Synth** — oscillator, voice, filter, envelope, and core tone architecture
2. **Perform** — macro bank, keyboard zone, per-note MPE monitoring, gesture control, and preset morphing
3. **Matrix** — source/destination routing, envelope and LFO visualization, live modulation debugging
4. **FX** — chorus, delay, reverb, drive, and performance-facing global processing

## Development Horizon

### Phase A — Instrument Shell
- Lock the workspace navigation model
- Group the prototype into stable views
- Expose enough destinations that routing reflects the engine accurately

### Phase B — Performance Center
- Expand macro system toward the target performance panel
- Strengthen the MPE monitor and per-note expression readouts
- Improve keyboard/per-note interaction fidelity

### Phase C — Visual Modulation Modules
- Add filter response views
- Add envelope and LFO visual modules
- Improve morph visualization and preset presentation

### Phase D — Premium Surface Refinement
- Move styling closer to the target concept
- Increase clarity, hierarchy, and performance ergonomics
- Reduce prototype clutter while preserving depth

## Working Rule

Every new change should be evaluated against the whole instrument:
- Does it move Zephyr toward the target shell?
- Does it improve performance playability?
- Does it make the synth clearer, not just bigger?
- Does it support the eventual premium interface direction?
